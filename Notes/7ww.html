7wwW1W2 Rules and Examples

Rule 1: MANOBS 12.3.10 or 12.3.10.3.3 (1)

  • When no significant weather has been observed within the preceding synoptic period, this group will ALWAYS be omitted.
  • This rule never changes.

Rule 2: MANOBS 12.3.10 or 12.3.10.3.3 (2)

  • When there is only one weather type, extending from the previous synoptic period and continuous throughout the current synoptic period, the codes for ww, W1, and W2 will ALWAYS be coded as to represent the same weather type.
  • I.e. 8088

Rule 3: MANOBS 12.3.10 or 12.3.10.3.3 (17)

  • When there is only one weather type, beginning some time during the current synoptic period and occurring at the time of observation, the codes for W1 and W2 will ALWAYS be coded the same as to represent the trend of the cloud throughout the current synoptic period.
  • I.e. 8022

Rule 4: MANOBS 12.3.10 or 12.3.10.3.3 (12)

  • When there is only one weather type, extending from the previous synoptic period and stopping prior to the hour preceding the synoptic observation, the code for ww will ALWAYS be coded as to represent the cloud opacity at the time of observation; and, the codes for W1 and W2 will ALWAYS be coded as to represent the same weather type.
  • I.e. 0288

Rule 5: MANOBS 12.3.10 or 12.3.10.3.3 (16)

  • When there is only one weather type, extending from the previous synoptic period and stopping within the hour preceding the synoptic observation, however, NOT occurring at the time of observation, the code for ww will ALWAYS be coded using codes 20-29; and, W1 and W2 will represent any further, different, significant weather; or, the trend of the cloud throughout the current synoptic period if no further significant weather was observed.
  • I.e. 2522, 2582, 2586

Rule 6: MANOBS 12.3.10 or 12.3.10.3.3 (13)

  • It is crucial to ALWAYS be aware of any remarks transmitted throughout the current synoptic period as they may have an effect of the synoptic coding.
  • Certain remarks must be coded.
  • I.e. 1022, 1322
